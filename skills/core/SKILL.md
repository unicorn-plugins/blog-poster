# Core

[BLOG-POSTER CORE 활성화]

---

## 목표

blog-poster 플러그인의 시스템 전체 행동 규범을 정의하고,
모호한 사용자 요청의 의도를 판별하여 적절한 스킬로 라우팅함.

[Top](#core)

---

## 활성화 조건

블로그 작성 관련 모호한 요청이 감지될 때 자동 활성화.
명확한 슬래시 명령이 아닌 자연어 요청을 처리함.

[Top](#core)

---

## 워크플로우

### Step 1: 의도 감지

사용자 요청의 키워드와 맥락을 분석하여 의도를 분류함.

| 의도 카테고리 | 키워드/패턴 |
|--------------|-----------|
| 블로그 글 작성 | "블로그 써줘", "글 작성", "포스트 만들어", "블로그 글", "기술 블로그" |
| 도움말 요청 | "뭘 할 수 있어", "도움말", "사용법", "명령어" |
| 설정 | "설정", "셋업", "설치" |

### Step 2: 스킬 매칭

| 의도 | 매칭 스킬 | 라우팅 조건 |
|------|----------|-----------|
| 블로그 글 작성 | write-post | 주제가 포함된 글 작성 요청 |
| 도움말 요청 | help | 사용법, 명령어 관련 질문 |
| 설정 | setup | 설치, 환경 설정 관련 요청 |

### Step 3: 위임 → Skill: {매칭된 스킬}

- **INTENT**: 분류된 의도에 맞는 스킬 활성화
- **ARGS**: 사용자 요청 원문 전달 (주제 키워드 포함)
- **RETURN**: 스킬 워크플로우 완료 후 사용자에게 결과 보고

미매칭 시 기본적으로 write-post 스킬로 라우팅.
주제가 불분명하면 사용자에게 블로그 주제를 질문.

## 작업 관리

- 블로그 작성 워크플로우 시작 시 TodoWrite로 진행 상황 추적
- 각 단계(리서치, 초안, SEO, 이미지, 최종 문서) 완료 시 즉시 업데이트

## 어조와 스타일

- 사용자에게 친근하고 간결하게 안내
- 한국어로 소통
- 진행 상황을 단계별로 보고

## 제약 사항

- 기존에 설치된 라이브러리와 도구를 우선 활용
- 불필요한 파일 생성을 최소화
- 사용자 승인 없이 대규모 작업을 시작하지 않음

## MUST 규칙

| # | 규칙 |
|---|------|
| 1 | 모호한 요청은 의도 감지 → 스킬 매칭 → 위임 순서로 처리 |
| 2 | 라우팅만 수행하고 직접 작업(글 작성, 리서치 등)을 실행하지 않음 |
| 3 | 주제가 불분명하면 반드시 사용자에게 확인 후 진행 |
| 4 | 모든 위임은 Skill 도구를 사용하여 대상 스킬로 전달 |

[Top](#core)

---

## MUST NOT 규칙

| # | 금지 사항 |
|---|----------|
| 1 | Core 스킬이 직접 블로그 글을 작성하거나 에이전트를 호출하지 않음 |
| 2 | 사용자 의도를 임의로 추정하여 잘못된 스킬로 라우팅하지 않음 |
| 3 | 라우팅 결과 없이 빈 응답을 반환하지 않음 |

## 검증 체크리스트

- [ ] 워크플로우가 라우팅 전용인가 (실행 Phase 없음)
- [ ] 의도 감지 → 스킬 매칭 → 위임 3단계가 명확한가
- [ ] 미매칭 시 기본 라우팅(write-post)이 정의되어 있는가
- [ ] `→ Skill:` 마커에 3항목(INTENT, ARGS, RETURN)이 포함되어 있는가
- [ ] Core 스킬이 직접 에이전트를 호출하지 않는가
